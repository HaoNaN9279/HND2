# 命运2 全量静态游戏数据库（无玩家数据，官方原版Manifest）

## 一、官方总入口（获取完整数据库下载地址，实时最新）

### 1\. Manifest 根接口（必须带 API Key 请求）

```Plain Text
https://www.bungie.net/Platform/Destiny2/Manifest/
```

作用：返回当前版本全量游戏定义库的下载链接，包含**SQLite 完整数据库文件**与分块 JSON 定义文件，涵盖武器、护甲、Perk、职业、地图、任务、成就、模组、剧情等全部游戏静态资源，完全不含任何玩家账号 / 角色数据。

### 请求要求

1. 先注册 Bungie 应用获取 API Key：[https://www\.bungie\.net/en/Application](https://www.bungie.net/en/Application)

2. 请求头必须携带 `X-API-Key: 你的密钥`

## 二、完整离线 SQLite 数据库地址（简体中文 zh\-chs）

调用上方 Manifest 接口后，返回 JSON 内 `mobileWorldContentPaths["zh-chs"]` 会给出实时完整库路径，拼接域名即可下载：

```Plain Text
https://www.bungie.net{接口返回的path字段}
```

示例拼接格式（路径随版本更新自动变化，不要硬写此示例）：

```Plain Text
https://www.bungie.net/common/destiny2_content/sqlite/zh-chs/world_sql_content_xxxxxx.content
```

- 文件是 ZIP 压缩包，解压后后缀改为 `.sqlite3`，可用 Navicat、DB Browser for SQLite 直接打开全量数据表。

- 包含所有 `Destiny*Definition` 表，是官方唯一完整离线数据库。

## 三、分表 JSON 独立地址（按需单表拉取）

Manifest 内 `jsonWorldComponentContentPaths` 存放单表 JSON 链接，例如装备定义表：

```Plain Text
https://www.bungie.net/common/destiny2_content/json/zh-chs/DestinyInventoryItemDefinition.json
```

## 四、补充说明

1. **Manifest = 静态全局游戏数据库**：仅装备、机制、地图等固定内容；`/Profile`、`/Character` 才是玩家数据，两者完全分离。

2. 版本更新（赛季 / 补丁）后 Manifest 地址会自动变更，必须重新请求 `/Destiny2/Manifest/` 获取新下载链接。
